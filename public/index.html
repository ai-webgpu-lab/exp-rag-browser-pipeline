Browser RAG Pipeline Harness

`exp-rag-browser-pipeline` now runs a deterministic browser-only ingest, chunk, embed, retrieve, rerank, and answer loop over a bundled fixture. The result surface is repo-specific even before a production embedder or answer model is wired in.

This baseline is intentionally transparent: answer generation is extractive and citation hit-rate is scored against expected source documents.

Pipeline Controls

Run the full pipeline to ingest bundled documents, generate deterministic embeddings, and answer the bundled question set with citations.

Harness Scope

  • Document ingestion, chunking, retrieval, and citation scoring happen in-browser.
  • Embedding and answer generation are deterministic stand-ins until a real runtime lands.
  • The bundled fixture keeps question count and expected citations stable across browsers.
